# AI Repository Analysis Prompt

Production-grade prompt for analyzing repositories and designing scalable, token-efficient Claude Code workflows.

This repository contains a single powerful prompt designed to help AI coding agents deeply analyze a codebase and generate an optimized operating model for long-term development.

The prompt is especially useful for:
- Large repositories
- Monorepos
- Microservice architectures
- Enterprise applications
- Long-term active projects
- Multi-developer teams

---

# What This Prompt Does

The prompt instructs Claude Code (or other advanced coding agents) to:

- Analyze repository architecture
- Understand project structure
- Detect high-context/problematic areas
- Design a scalable `CLAUDE.md`
- Build memory architecture
- Create rules and slash commands
- Optimize token usage
- Recommend hooks and workflows
- Detect AI workflow anti-patterns
- Design long-term AI-assisted development strategies

Instead of generic prompting, this approach focuses on:
- context engineering
- token efficiency
- maintainability
- scalable AI workflows

5. Review the generated recommendations

- actively maintained systems
- projects with complex architecture

# Recommended Workflow

For best results:

- Use focused sessions
- Avoid loading the entire repository unnecessarily
- Keep architectural memory structured
- Compact context regularly
- Split large systems into bounded areas
- Prefer reusable rules and commands
- Keep `CLAUDE.md` concise

---

# Common Anti-Patterns

Avoid:
- giant `CLAUDE.md` files
- analyzing the whole repository every session
- storing everything in memory
- long unstructured chats
- duplicated architecture descriptions
- context-heavy debugging logs
- oversized prompts
